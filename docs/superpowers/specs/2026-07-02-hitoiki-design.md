# ひといき 設計ドキュメント

作成日: 2026-07-02
ステータス: 承認済み(ブレインストーミング完了)

## 概要

「ひといき」は、視覚的な呼吸ガイドを通じて利用者が呼吸のペースを整えることを支援するWebアプリケーション。利用者は画面上のガイドに従うだけで利用でき、複雑な操作や判断を必要としない。

メインターゲットはパニック発作経験者・不安症状を抱える人。想定利用シーンは電車内・会議前など、静かな公共の場が中心。したがって**音声なしで完結すること**と**通信環境に依存しないこと**が設計の根幹となる。

本設計では特に**アクセシビリティ**と**オフライン対応**を重点項目とする。

## 決定事項

| 項目 | 決定 |
|---|---|
| 技術スタック | SvelteKit + adapter-static(完全静的サイト) |
| オフライン対応 | PWA化(Service Worker で全アセットを事前キャッシュ) |
| 時間設定 | プリセット + 詳細設定で各フェーズの秒数を自由調整 |
| フェーズ構成 | 内部4フェーズ(吸う→止める→吐く→止める)、0秒フェーズは自動スキップ |
| アニメーション駆動 | JSステートマシン + CSS transform(案A) |
| 設定の永続化 | localStorage |
| 言語 | 日本語のみ |
| 音声 | 使用しない |
| 振動 | オプション(デフォルトオフ、非対応端末ではトグル非表示) |
| デプロイ先 | 未定(静的ホスティングであれば何でも可。実装完了後に決定する) |

## アーキテクチャ

SvelteKit + adapter-static による完全静的PWA。サーバー・API・外部通信は一切持たない。ページは実質1画面(メイン画面)で、設定はモーダル/ボトムシート内で完結する。

- ビルド成果物は軽量を維持する(目標: 初回ロード合計 100KB 未満)
- 外部フォント・外部CDN・アナリティクス等の外部リソースは読み込まない(オフライン確実性と「通信環境に依存しない」要件のため)
- フォントはシステムフォントを使用する

## 画面構成

1画面主義。利用者の視線は中央の円に固定される。

- **中央**: 呼吸ガイド円。吸うで拡大、吐くで縮小、止めるは静止。円の縁に沿ってフェーズの残り時間を示す細い進捗リングを描く(色以外の時間手がかり)
- **円の中央**: フェーズ名テキスト「すって」「とめて」「はいて」。ひらがな・大きめの文字で、視線が円から動かない位置に置く
- **下部**: 開始/停止の大きなボタン1つ。片手操作時に親指で届く位置
- **右上**: 設定アイコン(プリセット選択・秒数調整・振動オンオフ)
- ガイド実行中は設定アイコン等を減光し、視界のノイズを最小化する

### プリセット

初回起動時はデフォルトプリセットが選択済みで、開始ボタンを押すだけで利用開始できる(「考えさせない」コンセプト)。

| プリセット | 吸う | 止める | 吐く | 止める |
|---|---|---|---|---|
| ゆっくり(デフォルト) | 4秒 | 0秒 | 6秒 | 0秒 |
| 4-7-8呼吸 | 4秒 | 7秒 | 8秒 | 0秒 |
| ボックス呼吸 | 4秒 | 4秒 | 4秒 | 4秒 |
| カスタム | 吸う・吐くは 1〜10秒、止めるは 0〜10秒で自由設定 |

デフォルトプリセットは「吐く方が長い」穏やかなパターンとし、息止めを含まない(息止めは不安時に苦しく感じる人がいるため)。

## コンポーネント分割

| ユニット | 責務 | 依存 |
|---|---|---|
| `BreathingEngine`(純TSモジュール) | フェーズ進行のステートマシン。`requestAnimationFrame` ベースで「現在フェーズ・フェーズ内経過率・残り秒数」を購読可能な形で公開。UIに依存しない。**単体テストの主対象** | なし |
| `BreathingCircle.svelte` | エンジンの状態を受けて円・フェーズテキスト・進捗リングを描画 | BreathingEngine の状態 |
| `ControlBar.svelte` | 開始/停止ボタン | なし(イベント発火のみ) |
| `SettingsSheet.svelte` | プリセット選択・秒数スライダー・振動トグル | settings |
| `settings.svelte.ts` | 設定の状態管理と localStorage 永続化 | なし |
| Service Worker | 全アセットの事前キャッシュとオフライン提供 | なし |

## データフロー

1. 起動時: localStorage から設定を読み込む(なければデフォルト)
2. 開始: 設定(各フェーズの秒数)を `BreathingEngine` に渡して開始
3. 実行中: エンジンが rAF でフェーズを進行し、UIが状態を購読して描画・振動を行う
4. 設定変更: 実行中の変更は**次サイクルの頭から**反映する(円が途中で飛ばないように)
5. 停止: エンジンを停止し、円を初期状態に戻す

## アクセシビリティ(重点項目)

状態の判別は常に複数の経路で伝える。**色は補助であり、色を知覚できなくても全機能が使える**こと。

### マルチモーダルな状態表現

各フェーズは以下の3経路+オプション1経路で判別できる:

1. **形**: 円の拡大(吸う)・縮小(吐く)・静止(止める)
2. **テキスト**: フェーズ名「すって」「とめて」「はいて」
3. **進捗リング**: フェーズ内の残り時間を形状で表示
4. **振動**(オプション): フェーズ切り替わり時に短い振動パターン(Vibration API)

### 具体的要件

- **色コントラスト**: WCAG 2.1 AA(テキスト 4.5:1、UIコンポーネント 3:1)を満たす。落ち着いた配色の中でもコントラストは確保する
- **prefers-reduced-motion**: 有効時はスケール変化を控えめ(または透明度変化に置換)にし、テキストと進捗リング主体の表現に切り替える
- **prefers-color-scheme**: ライト/ダーク両対応。暗い車内でも眩しくない
- **文字サイズ**: フェーズテキストは大きく表示し、ユーザーのブラウザ文字サイズ設定(rem基準)に追従する
- **タッチターゲット**: すべての操作要素は 44×44px 以上
- **スクリーンリーダー**: フェーズ変化を `aria-live="polite"` で通知。開始/停止ボタンには明確なラベル。装飾的なアニメーション要素は `aria-hidden`
- **キーボード操作**: 開始/停止・設定の全操作がキーボードのみで可能。フォーカスリングを消さない
- **音声なし**: 音は一切使わない(静かな環境での利用が前提)

## オフライン対応(重点項目)

「地下鉄・機内で開いたら確実に動く」ことを保証する。

- **Service Worker**: precache 方式。ビルド時に全アセット(HTML/JS/CSS/アイコン)をマニフェスト化し、初回訪問時にすべてキャッシュする。SvelteKit 標準の `$service-worker` モジュールで実装する
- **キャッシュ戦略**: cache-first。ネットワークには原則アクセスしない(更新チェックは Service Worker の標準更新機構に任せる)
- **更新**: 新バージョンの Service Worker はバックグラウンドで取得し、次回起動時に有効化。**利用中に強制リロードしない**(呼吸の途中で画面が変わることは絶対に避ける)
- **Web App Manifest**: ホーム画面追加に対応(`display: standalone`、テーマカラー、アイコン一式)
- **外部リソースゼロ**: フォント・画像・スクリプトすべて自己ホスト。オフラインで欠けるものが存在しない状態にする
- **検証**: オフライン動作(DevTools のネットワーク遮断)をリリース前チェックリストに含める

## そのほかの端末対応

- **Wake Lock API**: ガイド実行中は画面スリープを防止。非対応端末では単に効かないだけで壊れない
- **Vibration API**: iOS Safari 非対応のため、機能検出して非対応時はトグル自体を表示しない
- **低スペック端末**: アニメーションは `transform` と `opacity` のみ(GPU合成のみで完結し、レイアウトを発生させない)

## エラー処理

外部通信ゼロのため、エラー源はほぼ「ブラウザAPI非対応」のみ。方針は**すべて機能検出して静かにフォールバック**(利用者にエラーを見せない・壊れない)。

| エラー源 | フォールバック |
|---|---|
| localStorage 不可(プライベートモード等) | デフォルト設定でメモリ上のみ動作 |
| Wake Lock 非対応 | 何もしない(スリープ防止なしで動作) |
| Vibration 非対応 | 振動トグルを非表示 |
| Service Worker 非対応 | 通常のWebサイトとして動作 |

## テスト

- **`BreathingEngine` のユニットテスト(Vitest)**: フェーズ遷移、0秒フェーズのスキップ、途中停止(初期状態へ戻ること)、設定変更が次サイクル頭から反映されること
- **設定の永続化テスト**: 保存・読み込み・localStorage 不可時のフォールバック
- **コンポーネントテスト**: 主要な描画分岐(フェーズごとのテキスト・reduced-motion 分岐)の smoke テスト。Svelte MCP ツールでコード検証を併用
- **手動チェックリスト**: オフライン動作、スクリーンリーダー読み上げ、キーボード操作、コントラスト、ライト/ダーク表示

## スコープ外(YAGNI)

- 利用履歴・統計・記録機能
- アカウント・ログイン
- 通知・リマインダー
- 多言語対応
- 音声ガイド
- ネイティブアプリ化

## 制約事項(仕様書より)

- 医療行為を目的としない。診断や治療を行うものではない
- 利用者の体調変化について保証を行わない
- 医療機器的な見た目・病気を強調する表現・危機感を煽る表現を避ける(ブランドイメージ: 安心・静けさ・シンプル・そっと支える)
